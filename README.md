# Gemma 4 on Android
Private, Offline AI — No API Keys Required

Android inventory app powered by **Gemma 4 E2B** running completely offline.

The user types the supplier's text in natural language and Gemma extracts, classifies and structures the products — no internet, no server, no API key.

---

## Stack

- **Kotlin + Jetpack Compose**
- **LiteRT-LM** (`com.google.ai.edge.litertlm:litertlm-android:0.10.0`)
- **Gemma 4 E2B** (`.litertlm` format)
- **kotlinx.serialization** to parse the JSON generated by the model

## Architecture

```
data/ai/        → LocalInferenceEngine (interface) + LiteRtInferenceEngine
data/litert/    → LocalInventoryRepository (prompt + inference + parsing)
data/util/      → ModelCopier (copies assets/ → filesDir)
domain/model/   → InventoryItem
presentation/   → InventoryViewModel + SmartInventoryScreen
```

---

## Setup

### 1. Download the model

The model is **not included in the repository** due to its size (~2.4 GB).

Download it from Hugging Face:
```
https://huggingface.co/litert-community/Gemma4-4B-IT
```
File: `gemma-4-E2B-it.litertlm`

### 2. Place the model

```
app/src/main/assets/
└── gemma-4-E2B-it.litertlm
```

### 3. Build and run

```bash
./gradlew :app:installDebug
```

The first launch copies the model to internal storage (~20–60 sec). Subsequent launches are instant.
